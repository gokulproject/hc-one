================================================================================
RULE #0 — IST EVERYWHERE (READ THIS FIRST)
================================================================================

The MySQL server clock runs in UTC. We will override this at the connection
level so every NOW() call inside MySQL returns IST, and every datetime Python
writes to DB is a plain naive IST datetime.

THE SINGLE RULE:
  - Every datetime stored in MySQL = naive IST datetime (no tzinfo).
  - Every datetime read from MySQL = already IST, display as-is.
  - Never store UTC in any HC table column.
  - USE_TZ = False in settings (disables Django's UTC-aware datetimes).
  - TIME_ZONE = "Asia/Kolkata" in settings.
  - MySQL connection init_command forces: SET time_zone = '+05:30'
    so MySQL's NOW() / NOW(3) returns IST automatically.

CONSEQUENCE:
  - `datetime.now()` in Python → IST (because TIME_ZONE = Asia/Kolkata,
    USE_TZ = False means Django uses local time = IST if OS is IST, BUT
    to be safe always use `now_ist()` from ist_utils.py instead of
    `datetime.now()` or `datetime.utcnow()` anywhere in scheduler code).
  - `NOW()` in raw MySQL queries → IST (because of SET time_zone = '+05:30').
  - DB fields like last_run_at, next_run_at, scheduled_fire_at,
    actual_start_at, completed_at, logged_at, scheduled_at → all IST.
  - Admin panel shows DateTimeField values → IST (no conversion needed).
  - UI templates that call utc_to_ist() → REMOVE those conversions,
    display the value directly as it comes from DB (it is already IST).

================================================================================
RULE #1 — DJANGO-Q RUNS INSIDE THE APP (NO SEPARATE PROCESS)
================================================================================

Django-Q normally requires a separate `python manage.py qcluster` process.
We do NOT want that. We want Django-Q's task queue to run inside the Django
app process itself, triggered by APScheduler's replacement logic in AppConfig.

HOW TO ACHIEVE THIS:
  Use Django-Q in SYNC mode for the broker + run a lightweight in-process
  thread that acts as the cluster. Specifically:

  Option: Use `django_q.tasks.async_task` with `sync=True` in Q_CLUSTER,
  which makes every async_task() call execute immediately in the same thread.
  This is the simplest approach — no worker process, no Redis, no ORM broker
  overhead. Tasks run synchronously in a background thread we manage.

  IMPLEMENTATION PATTERN:
  - Q_CLUSTER sync=True means async_task() runs the function immediately
    in the calling thread (no queue, no worker).
  - Schedule checking (tick every minute) is done by a single daemon thread
    started in AppConfig.ready() — exactly like APScheduler's background
    thread but without the APScheduler library.
  - Manual triggers (run_now, rerun) spawn a daemon thread directly
    (same as before) since sync=True means async_task would block the
    request thread.

  SO THE FINAL ARCHITECTURE IS:
    - One background daemon thread (started in AppConfig.ready()) that
      wakes every 60 seconds, checks schedules due to fire, and launches
      them in sub-threads.
    - async_task() used only for task registration pattern (or replaced
      with direct threading for manual runs).
    - django_q installed but used primarily for its Schedule model and
      ORM-based task tracking — NOT for its cluster worker process.

  SIMPLER ALTERNATIVE (RECOMMENDED):
  Since we want in-process and no external worker, do the following:
    1. Install django-q2 (for its models, admin, tracking).
    2. Set Q_CLUSTER sync=True so no worker needed.
    3. Replace APScheduler's BackgroundScheduler with a plain Python
       threading.Timer loop (60-second tick) started in AppConfig.ready().
    4. Use django_q.tasks.async_task() for fire-and-forget task calls
       (they will run in-thread synchronously due to sync=True).
    5. This means: zero external processes, zero Redis, zero ORM polling.

================================================================================
RULE #2 — SCHEDULED RUN FIRES AT EXACT start_at TIME (ZERO DELAY)
================================================================================

CURRENT PROBLEM with APScheduler:
  APScheduler registers a CronTrigger and fires the job at the next cron tick.
  If start_at = 04:43 PM IST and cron = "43 16 * * *", it fires at 04:43 PM.
  But if the scheduler starts at 04:44 PM, that tick is missed.

REQUIRED BEHAVIOUR:
  - When the tick loop wakes up each minute, it checks every active schedule.
  - For each schedule, it fires if ALL of these are true:
      a. is_active = 1 AND customer is_active = 1
      b. start_at has been reached (now_ist >= start_at_ist)
      c. The current minute matches the cron expression (croniter says it's due)
      d. last_run_at is None OR last_run_at < this cron tick's due time
         (prevents double-fire within the same minute)

  - ZERO DELAY: The tick loop sleeps for exactly (60 - current_seconds) so it
    wakes at the top of each minute (e.g., 04:43:00). Do NOT use time.sleep(60)
    blindly — it drifts. Use this pattern:

      import time
      from datetime import datetime
      while running:
          now = datetime.now()  # IST (USE_TZ=False, TIME_ZONE=Asia/Kolkata)
          sleep_secs = 60 - now.second - now.microsecond / 1_000_000
          time.sleep(max(sleep_secs, 0))
          _do_tick()

  - This ensures the tick fires at :00 of each minute, matching cron exactly.

    """Return current IST time as naive datetime."""

def next_run_from_start_at(cron_expression: str,
                            start_at_ist: datetime) -> datetime:
    """
    Given start_at in IST, compute the first cron fire time.

# ── Load schedules from DB ────────────────────────────────────────────────────
def _load_schedules() -> list:
    try:
        eng = _get_engine()
        with eng["DBManager"]() as db:
            return db.fetch_all(
                """SELECT s.id, s.customer_id, s.env_types, s.cron_expression,
                          s.schedule_name, s.start_at, s.last_run_at, s.next_run_at,
                          c.name AS customer_name
                   FROM schedules s
                   JOIN customers c ON c.id=s.customer_id
                   WHERE s.is_active=1 AND c.is_active=1""")
    except Exception as exc:
        logger.error(f"Cannot load schedules: {exc}")
        return []


# ── Tick: check all schedules and fire due ones ───────────────────────────────
def _do_tick():
    """
    Called every minute (aligned to :00 of each minute).
    Checks all active schedules and fires those that are due.

    Due condition:
      1. start_at reached (or no start_at set)
      2. Current minute matches cron expression (croniter confirms it)
      3. Not already run this cron tick (last_run_at < current cron due time)
    """
    from croniter import croniter

    schedules = _load_schedules()
    now       = now_ist()

    # Strip seconds so we compare at minute precision
    now_minute = now.replace(second=0, microsecond=0)

    for sched in schedules:
        try:
            cron_expr = sched["cron_expression"]
            last_run  = sched.get("last_run_at")

            # Compute what the current cron tick is (the minute that just fired)
            # Use now_minute - 1 second as base so croniter returns this minute
            ci = croniter(cron_expr, now_minute - timedelta(seconds=1))
            due_at = ci.get_next(datetime).replace(second=0, microsecond=0)

            # Is the current minute exactly the due minute?
            if due_at != now_minute:
                continue  # Not due this minute

            # Has this tick already been fired? (last_run_at >= due_at)
            if last_run is not None:
                if hasattr(last_run, 'tzinfo') and last_run.tzinfo is not None:
                    last_run = last_run.replace(tzinfo=None)
                last_run_minute = last_run.replace(second=0, microsecond=0)
                if last_run_minute >= due_at:
                    continue  # Already fired for this tick

            # Fire this schedule
            _fire_schedule(sched)

        except Exception as exc:
            logger.error(
                f"[Scheduler] Tick error for schedule id={sched.get('id')}: {exc}",
                exc_info=True)

                try:
                    email_svc = EmailService(db, SilentLogger())
                    for et in [e.strip() for e in env.split(",")]:
                        email_svc.send_failure(
                            run_id=run_id,
                            customer_id=run["customer_id"],
                            customer_name=cname,
                            env_type=et,
                            error_message=fail_reason,
                            triggered_by=run.get("triggered_by", "system"),
                            step_failures=[{
                                "step": "Run Recovery",
                                "reason": fail_reason}])
                except Exception as exc:
                    logger.error(
                        f"[Scheduler] Failed to send stuck-run email for #{run_id}: {exc}")

                # Queue rerun
                try:
                    cur_att = run.get("rerun_attempt") or 0
                    mx      = cfg.rerun_max_attempts
                    nxt_att = cur_att + 1
                    if nxt_att <= mx:
                        # Schedule rerun: IST datetime stored directly
                        sched_at = now_ist() + timedelta(hours=cfg.rerun_interval_hours)
                        existing = db.fetch_one(
                            "SELECT id FROM rerun_queue "
                            "WHERE original_run_id=%s "
                            "AND status IN ('PENDING','IN_PROGRESS')",
                            (run_id,))
                        if not existing:
                            db.insert_get_id(
                                """INSERT INTO rerun_queue
                                   (original_run_id, customer_id, env_types,
                                    attempt_number, max_attempts,
                                    status, scheduled_at, last_error)
                                   VALUES (%s,%s,%s,%s,%s,'PENDING',%s,%s)""",
                                (run_id, run["customer_id"], env,
                                 nxt_att, mx, sched_at, fail_reason[:2000]))
                            logger.info(
                                f"[Scheduler] {cname}/{env} | Queued recovery rerun "
                                f"attempt {nxt_att}/{mx}")
                    else:
                        db.execute(
                            """UPDATE rerun_queue
                               SET status='EXHAUSTED', last_error=%s, completed_at=NOW()
                               WHERE original_run_id=%s
                                 AND status NOT IN ('EXHAUSTED','COMPLETED')""",
                            (fail_reason[:500], run_id))
                        logger.info(
                            f"[Scheduler] {cname}/{env} | Run #{run_id} "
                            f"exhausted all {mx} attempts (recovered)")
                except Exception as exc:
                    logger.error(
                        f"[Scheduler] Failed to queue rerun for stuck run #{run_id}: {exc}")
    except Exception as exc:
        logger.error(f"[Scheduler] _recover_stuck_runs error: {exc}", exc_info=True)


# ── Main scheduler thread loop ────────────────────────────────────────────────
def _scheduler_loop():
    """
    Background daemon thread. Ticks every minute aligned to :00.
    Checks schedules, fires due ones, and runs stuck-run recovery every 10 ticks.
    """
    global _scheduler_running
    logger.info("[Scheduler] Thread started (Django-Q in-process mode).")

    tick_count = 0
    while _scheduler_running:
        # Sleep until the top of the next minute (zero drift)
        now        = now_ist()
        sleep_secs = 60 - now.second - now.microsecond / 1_000_000
        time.sleep(max(sleep_secs, 0.1))

        if not _scheduler_running:
            break

        tick_count += 1
        ist_now = now_ist()
        logger.info(
            f"[Scheduler] Tick #{tick_count} | "
            f"{ist_now.strftime('%d-%b-%Y %I:%M:%S %p IST')}")

        try:
            _do_tick()
        except Exception as exc:
            logger.error(f"[Scheduler] Tick error: {exc}\n{traceback.format_exc()}")

        # Stuck-run recovery every 10 minutes (every 10 ticks)
        if tick_count % 10 == 0:
            try:
                _recover_stuck_runs()
            except Exception as exc:
                logger.error(f"[Scheduler] Stuck-run recovery error: {exc}")

    logger.info("[Scheduler] Thread stopped.")


# ── Lifecycle ─────────────────────────────────────────────────────────────────
def start_scheduler():
    """Start the background scheduler thread. Called from AppConfig.ready()."""
    global _scheduler_thread, _scheduler_running
    with _scheduler_lock:
        if _scheduler_thread and _scheduler_thread.is_alive():
            logger.info("[Scheduler] Already running.")
            return
        _scheduler_running = True
        _scheduler_thread  = threading.Thread(
            target=_scheduler_loop,
            daemon=True,
            name="hc_scheduler")
        _scheduler_thread.start()
        logger.info("[Scheduler] Started (in-process, Django-Q sync mode).")


def stop_scheduler():
    global _scheduler_running
    with _scheduler_lock:
        _scheduler_running = False
    logger.info("[Scheduler] Stop signal sent.")


def restart_scheduler():
    stop_scheduler()
    time.sleep(2)
    start_scheduler()


def get_scheduler():
    return _scheduler_thread


# ── scheduler_status() — for UI dashboard ────────────────────────────────────
def scheduler_status() -> dict:
    """
    Returns scheduler status for the dashboard/UI.
    All datetime values come from DB as IST — display directly, no conversion.
    """
    now_ist_str = now_ist().strftime("%d-%b-%Y %I:%M:%S %p")
    is_running  = bool(_scheduler_thread and _scheduler_thread.is_alive())

    if not is_running:
        return {
            "running":          False,
            "job_count":        0,
            "jobs":             [],
            "active_runs":      [],
            "active_run_count": 0,
            "timezone":         "Asia/Kolkata",
            "server_time_ist":  now_ist_str,
        }

    # Load schedules to show as "jobs" in the UI
    schedules = _load_schedules()
    jobs = []
    for s in schedules:
        nxt = s.get("next_run_at")
        # next_run_at is already IST (DB is IST-native)
        if nxt:
            if hasattr(nxt, 'tzinfo') and nxt.tzinfo is not None:
                nxt = nxt.replace(tzinfo=None)
            nxt_str = nxt.strftime("%d-%b-%Y %I:%M %p IST")
        else:
            nxt_str = "—"
        jobs.append({
            "id":       f"sched_{s['id']}",
            "name":     s["schedule_name"],
            "next_run": nxt_str,
        })

    with _active_runs_lock:
        active = [{"run_id": rid, "label": lbl}
                  for rid, lbl in _active_runs.items()]

    return {
        "running":          True,
        "job_count":        len(jobs),
        "jobs":             jobs,
        "active_runs":      active,
        "active_run_count": len(active),
        "timezone":         "Asia/Kolkata",
        "server_time_ist":  now_ist_str,
    }

def run_now_sync(schedule_id: int) -> Optional[int]:
    """Admin 'Run Now'. Spawns daemon thread. All DB writes in IST."""
    eng = _get_engine()
    try:
        with eng["DBManager"]() as db:
            eng["ConfigLoader"].load(db)
            s = db.fetch_one(
                "SELECT s.*, c.name AS cname FROM schedules s "
                "JOIN customers c ON c.id=s.customer_id WHERE s.id=%s",
                (schedule_id,))
            if not s:
                return None

            now      = now_ist()   # IST
            next_ist = next_cron_after(s["cron_expression"], now)  # IST

            run_id = db.insert_get_id(
                """INSERT INTO execution_runs
                   (schedule_id,customer_id,env_types,
                    run_type,status,triggered_by)
                   VALUES(%s,%s,%s,'MANUAL','PENDING','admin:run_now')""",
                (s["id"], s["customer_id"], s["env_types"]))

            db.execute(
                "UPDATE schedules SET last_run_at=%s, next_run_at=%s WHERE id=%s",
                (now, next_ist, schedule_id))   # IST values

            tracker_id = _create_tracker(
                db, schedule_id, s["customer_id"], s["env_types"],
                s["schedule_name"], s["cron_expression"],
                fire_at_ist=now,
                triggered_by="admin:run_now")

        label = f"{s['cname']}/{s['env_types']}"
        t = threading.Thread(
            target=_execute_run,
            args=(run_id, label, tracker_id),
            daemon=True,
            name=f"hc_run_{run_id}")
        t.start()
        return run_id
    except Exception as exc:
        logger.error(f"run_now_sync: {exc}", exc_info=True)
        return None


def run_customer_now(customer_id: int, env_types: str) -> Optional[int]:
    """Manual run for customer+env. Spawns daemon thread."""
    eng = _get_engine()
    try:
        with eng["DBManager"]() as db:
            eng["ConfigLoader"].load(db)
            run_id = db.insert_get_id(
                """INSERT INTO execution_runs
                   (customer_id,env_types,run_type,status,triggered_by)
                   VALUES(%s,%s,'MANUAL','PENDING','admin:manual')""",
                (customer_id, env_types))
            cust  = db.fetch_one(
                "SELECT name FROM customers WHERE id=%s", (customer_id,))
            cname = cust["name"] if cust else str(customer_id)

        label = f"{cname}/{env_types}"
        t = threading.Thread(
            target=_execute_run,
            args=(run_id, label, 0),
            daemon=True,
            name=f"hc_run_{run_id}")
        t.start()
        return run_id
    except Exception as exc:
        logger.error(f"run_customer_now: {exc}", exc_info=True)
        return None
